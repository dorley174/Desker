

# Desker — План реализации фронтенда

## Обзор
Desker — веб-приложение для бронирования рабочих мест в офисе. Сотрудники могут просматривать карту этажей, фильтровать места по оснащению, бронировать на выбранные часы, а также управлять своими бронированиями. Реализуем полный фронтенд с заглушками для бэкенда (Supabase).

---

## 1. Общий шаблон (Layout)
- **Header**: логотип «Desker» (кликабельный, возврат на главную), справа — кнопки авторизации или меню профиля
- **Footer**: год, ссылка на сайт, кнопка «О компании», контакты — единый для всех страниц
- Адаптивная вёрстка (десктоп + мобильные)

## 2. Главная страница — неавторизованный пользователь
- Приветственный лендинг-экран с описанием сервиса
- Кнопки «Войти» и «Зарегистрироваться» в хедере
- Заглушка-контент в main-блоке (преимущества сервиса, CTA)

## 3. Страницы авторизации
- **Вход**: форма email + пароль, ссылка «Забыли пароль?»
- **Регистрация**: форма с email, именем, фамилией, паролем и (опционально) инвайт-кодом
- **Сброс пароля**: заглушка формы
- Все формы с валидацией, заглушки для API-вызовов с комментариями

## 4. Главная страница — авторизованный пользователь (Бронирование)
Основной рабочий экран, как на вайрфреймах:
- **Левая панель**: список этажей (1–6), клик = выбор этажа
- **Панель фильтров/тегов**: чекбоксы оснащения (Принтер, Доска, 2+ мониторов, ТВ-панель и др.), фильтр статуса (Свободно / Занято)
- **Центральная область — карта**: визуализация зон (A, B, C, D…) с рабочими местами в виде пронумерованных ячеек. Цветовая индикация статуса: свободно, забронировано (другим), забронировано (мной), недоступно
- **Выбор даты**: датапикер над картой для выбора дня бронирования
- **Панель бронирования** (появляется при клике на место): название места, доступные часовые слоты с цветовой индикацией (свободен / занят), кнопки подтверждения/отмены
- **Профильное меню** в хедере: аватар + имя, выпадающий список (Профиль, Настройки, История, Выйти)

## 5. Страница «Настройки»
- Кнопка «Назад» (возврат на предыдущую страницу)
- Поля: смена имени, смена фамилии, смена пароля, смена аватара (загрузка файла)
- Кнопка «Сохранить изменения» — появляется только при наличии изменений
- Кнопка «Выйти из аккаунта»

## 6. Страница «История»
- Заголовок «Предыдущие брони»
- Список карточек бронирований: этаж, номер места, дата, время начала — время окончания, статус (завершена / отменена / no-show)
- Кнопка «Назад»

## 7. Страница «Профиль»
- Просмотр данных пользователя (имя, email, роль)
- Кнопка «Назад»

## 8. Моковые данные и заглушки
- Все данные (этажи, места, бронирования, пользователь) — из локальных моков с `// TODO: replace with API call` комментариями
- Имитация состояний авторизации через React Context
- Подготовленная структура для подключения Supabase (auth, database, RLS)

## 9. Инструкции по запуску
В README будут добавлены инструкции:
- Локальный запуск (`npm install` + `npm run dev`)
- Деплой на Vercel (подключение GitHub-репо, настройка)

---

## Стек
React + TypeScript + Tailwind CSS + shadcn/ui + React Router. Без бэкенда на этом этапе — только фронтенд с заглушками.
